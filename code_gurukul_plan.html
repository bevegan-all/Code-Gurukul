🔴 1. Problem Statement

The challenges faced by educational institutions running programming labs.

🚫

Lack of Lab Control

Students easily switch to browsers, open ChatGPT, or copy code from peers during lab sessions. Standard computer labs provide full OS access with no lockdown mechanism, defeating the purpose of practical assessments.

⏱️

Manual Grading Bottleneck

A single teacher managing 60+ students cannot evaluate hundreds of code submissions in real-time. Manual grading is slow, inconsistent, and introduces human bias. By the time feedback reaches students, the learning window has passed.

👁️

Zero Real-Time Visibility

Teachers have no way to see what each student is actually typing live. There is no system to detect idle students, students struggling silently, or detect when a student has Alt+Tabbed to an unauthorized application.

🛠️

IDE Fragmentation

Students use different IDEs (VS Code, Dev-C++, Turbo C) with varying compilers, creating an inconsistent environment. Setting up compilers on individual lab machines is tedious, error-prone, and wastes valuable class time.

🟢 2. Our Solution

A dual-platform approach that addresses every challenge.

CodeGurukul is a three-part system:


🌐

Cloud Web Portal

Hosted on Vercel, accessible from any browser. Provides separate dashboards for Admins (manage users, classes, departments) and Teachers (create assignments, quizzes, notes, and view live monitoring).

🖥️

Secure Windows App

A compiled Electron desktop application that runs in full kiosk mode. Blocks Alt+Tab, Task Manager, and all shortcuts. Contains a built-in Monaco Code Editor with multi-language local compilation.

⚙️

Backend Server

A Node.js/Express server running on a school PC. Connected to PostgreSQL and MongoDB databases locally, and exposed to the internet via ngrok tunneling for remote access.

✨ 3. Features Offered

Everything CodeGurukul provides out of the box.

👨‍💼 Admin Features

  • Create, edit, and disable Teachers and Students with welcome email notifications
  • Manage Departments → Courses → Classes → Subjects hierarchy
  • Assign teachers to subjects (major & minor) per class
  • View system-wide leaderboard with rankings
  • Password reset via 6-digit OTP email verification

👩‍🏫 Teacher Features

  • Create Lab Assignments with multiple sets and questions per subject/class
  • Create Quizzes with MCQ support (single/multiple correct answers)
  • Write and publish rich HTML Notes for students
  • Live Monitor Dashboard: See every student online/offline in real-time
  • Screen Streaming: Click on any student to view their live screen
  • View AI-graded submissions with override capability
  • Subject-wise leaderboard with automatic ranking

🧑‍🎓 Student Features (Windows App)

  • Login via secure Kiosk desktop application (blocks all system shortcuts)
  • View assigned lab assignments and open the Lab Workspace (Monaco IDE)
  • Write, compile, and test code locally in 5 languages (C, C++, Java, Python, Node.js)
  • Submit code for instant AI grading via Google Gemini
  • Take timed quizzes with auto-submit
  • Access teacher-published Notes
  • Use the Gemini Guru AI Tutor for context-aware coding help (without revealing answers)
  • Free Sandbox mode for open-ended practice and exploration
  • View personal Leaderboard with class rankings

🤖 AI & Automation Features

  • AI Auto-Grading: Compares student code against teacher's expected logic using Google Gemini, assigns a score out of 10 with a reason
  • Gemini Guru Chatbot: Context-aware conversational AI that guides students without directly providing answers; tracks chatbot usage count per submission
  • Multi-Key Rotation: Supports up to 4 Gemini API keys, auto-rotates on rate-limit (429) errors
  • Model Fallback Chain: Gracefully falls back through gemini-2.5-flash → 2.5-flash-lite → 2.0-flash → 1.5-flash

🟦 Google Technologies Used

Google Gemini API

Powering the automated AI grading engine — evaluates the correctness of student code submissions against the teacher's expected logic and assigns a score from 0-10 with contextual feedback.

Powering the Gemini Guru AI Tutor — a context-aware chatbot accessible in the student IDE via Ctrl+G to receive guided hints without direct answers.

Powers AI student reports — auto-generates a summary paragraph about each student's performance and areas of improvement.

Uses models: gemini-2.5-flash, gemini-2.5-flash-lite, gemini-2.0-flash, gemini-1.5-flash with auto-fallback.

Google V8 JavaScript Engine

Powers both the Node.js backend server and the Electron/Chromium runtime that drives the secure Windows desktop application. V8 is the underlying engine that makes the entire system run.

📦 Databases

PostgreSQL 15MongoDB

PostgreSQL stores all relational data (users, classes, assignments, grades). MongoDB stores flexible documents (notes, chatbot sessions, notifications, idle alerts).

🚀 Deployment

VercelngrokGitHub

Frontend hosted free on Vercel. Backend runs locally on a school PC and is exposed via ngrok tunnel. Source code versioned on GitHub with sensitive data in .gitignore.

🖥️ Compilers (Local)

GCC / G++PythonNode.jsjavac / java

All compilers are installed on the server machine. Student code is written to a temp sandbox, compiled/run via child_process.spawn, and output is streamed via WebSocket.

📊 Why Two Databases?

PostgreSQL is used for structured, relational data that requires ACID transactions, foreign key constraints, and complex JOINs (users, grades, assignments, class hierarchies).

MongoDB is used for flexible, document-oriented data where the schema can vary (rich HTML notes, variable-length chatbot conversations, nested message arrays).

🛡️ 17. Security Measures

How CodeGurukul ensures a secure, controlled environment.

🔒

Kiosk Lockdown

Electron app runs in fullscreen kiosk mode. Blocks Alt+Tab, Ctrl+Esc, Ctrl+Shift+Esc (Task Manager), and Alt+F4. The app is always-on-top and frameless.

🔑

JWT Authentication

All API routes are protected by JWT tokens. Tokens include role-based claims. Middleware validates tokens and enforces role guards (admin, teacher, student).

🔐

Secrets Management

All sensitive keys (DB credentials, JWT secret, Gemini API keys, email passwords) are stored in .env files which are .gitignore-d. Only .env.example templates are committed to GitHub.

🧹

Sandboxed Compilation

Student code is written to isolated /tmp/codegurukul_compiler/{uuid}/ directories. Each execution gets its own sandbox. Files are cleaned up after execution. A 60-second timeout kills runaway processes.

🔗

CORS & Helmet

Backend uses Helmet.js for HTTP security headers and CORS configured to allow cross-origin requests from the Vercel frontend and Electron app via ngrok.

📧

OTP Verification

Password resets require a 6-digit OTP sent via email (Nodemailer + Gmail SMTP). OTP codes expire after 10 minutes and are single-use.
